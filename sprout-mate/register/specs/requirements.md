# 需求说明

## 版本

register-1.0

## 背景

当前项目已有登录页和 `group` 多租户数据模型，但缺少新用户注册与分组初始化流程。

现状中，微信登录或账号登录都只支持已有用户。新用户无法自行创建成长空间，也无法通过安全的方式加入已有分组。系统中大量业务数据已经依赖 `group_id` 隔离，因此注册流程必须先确定用户所属分组。

## 核心目标

新增注册与人员管理能力，让新用户可以：

- 创建一个新的分组，成为该分组管理员。
- 使用管理员生成的一次性邀请码加入已有分组，默认成为普通用户。

让管理员可以：

- 在「我的」页面进入人员管理。
- 查看当前分组下所有成员。
- 调整成员权限为 `ADMIN` 或 `USER`。
- 生成一次性邀请码并分享给其他人。

## 问题描述

### 问题 1：缺少注册入口

- **现状**：项目只有登录页，登录接口只查找已有用户。
- **影响**：新用户无法进入系统，微信新用户登录时会出现「登录用户不存在」。

### 问题 2：缺少分组初始化流程

- **现状**：系统已有 `group` 表和 `user.group_id`，但没有从客户端创建分组的流程。
- **影响**：新家庭或新成长空间必须手动写数据库才能开始使用。

### 问题 3：加入分组不能暴露 `group_id`

- **现状**：如果让用户输入 `group_id` 加入分组，邀请码会变成 `1`、`2` 这类可枚举 ID。
- **影响**：体验生硬，也可能被用户随便尝试后加入不属于自己的分组。

### 问题 4：缺少成员和权限管理

- **现状**：`role` 已有 `ADMIN` 和 `USER`，但管理员没有管理成员权限的页面。
- **影响**：分组创建后无法邀请成员，也无法调整成员权限。

## 期望效果

| 场景 | 期望行为 |
|------|---------|
| 新用户创建分组 | 在注册页选择「创建分组」，输入分组名后创建 `group` 和当前用户，当前用户角色为 `ADMIN` |
| 新用户加入分组 | 在注册页选择「加入分组」，输入邀请码后加入邀请码对应的 `group`，当前用户角色为 `USER` |
| 注册成功 | 服务端返回 token，客户端完成登录态初始化并进入首页 |
| 管理员查看成员 | 在「我的」页面看到「人员管理」入口，进入后列出当前分组所有成员 |
| 管理员调整权限 | 管理员可将成员设置为 `ADMIN` 或 `USER` |
| 管理员生成邀请码 | 管理员生成一个一次性邀请码，分享给待加入用户 |
| 邀请码被使用 | 用户注册成功后，该邀请码失效，不允许再次使用 |
| 非管理员访问管理页 | 不展示入口；直接访问接口时服务端返回无权限 |
| 页面视觉与交互 | 注册页、人员管理页复用现有组件、状态展示和主题 token，不引入独立 UI 风格 |

## 涉及页面

| 页面 | 路径 | 职责 |
|------|------|------|
| 登录页 | `/login` | 新增「注册」入口，引导新用户创建或加入分组 |
| 注册页 | `/register` | 支持创建分组和通过邀请码加入分组 |
| 我的 | `/mine` | 管理员入口，新增「人员管理」菜单项 |
| 人员管理 | `/memberManage/entrance` | 查看成员、调整权限、生成邀请码 |

## 前端一致性要求

### 组件复用

新增页面必须优先复用现有 UI 组件和容器，避免重新实现已有能力。

| 场景 | 优先复用 |
|------|----------|
| 页面布局 | `Layout` |
| 管理列表 | `ConfigListPanel` |
| 加载、空状态 | `StatusWrapper` 或由 `ConfigListPanel` 内部承接 |
| 弹层 | `FloatLayout` |
| 表单项 | `FormItem` |
| 输入 | `Input`、`Textarea`、`PickerSelector` |
| 按钮 | `Button` |
| 间距 | `WhiteSpace` |
| 底部安全区 | `SafeAreaBar` |
| 图标 | `Icon` / iconfont 现有图标 |

### 样式 token

- SCSS 必须 `@import '@ui/styles/index.scss';` 后使用现有变量、函数和 mixin。
- 颜色优先使用 `$--color-*`、`$--text-color-*`、`$--fill-*`、`$--color-border-*` 等 token。
- TypeScript 内联颜色优先使用 `COLOR_VARIABLES`。
- 间距和尺寸使用 `px()`。
- 卡片、页面背景、阴影等优先使用现有 mixin，例如 `soft-page-background`、`glass-card`。
- 不允许在新增样式中随意写死 hex 颜色、rgba 阴影、圆角和背景，除非是明确新增设计 token 或临时兼容第三方组件。

### 人员管理页一致性

人员管理页应参考「我的」页面其他管理员入口页，而不是做成独立风格页面。

参考页面：

- `toy-taro-client/src/ui/pages/checkInManage/entrance/index.tsx`
- `toy-taro-client/src/ui/pages/prizeManage/entrance/index.tsx`
- `toy-taro-client/src/ui/pages/taskManage/categoryManage/index.tsx`

人员管理页要求：

- 使用 `ConfigListPanel` 承载成员列表。
- 使用 `loading` 透传给 `ConfigListPanel`，由其内部 `StatusWrapper` 展示 loading 和 empty 状态。
- 如某个区域不能使用 `ConfigListPanel`，必须显式使用 `StatusWrapper`，保持 `loading`、`count`、`loadingIgnoreCount` 和 `size` 行为与现有页面一致。
- 使用 `useSyncOnPageShow` 提供下拉刷新或页面返回刷新能力。
- 邀请码生成、角色编辑等操作使用现有 `Button` loading 状态或 `FloatLayout` 弹层，不新增自定义 loading UI。

## 数据模型变更

### 新增 group_invite 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT | 主键 |
| `group_id` | INT | 邀请码所属分组 |
| `code_hash` | VARCHAR(255) | 邀请码哈希值，不保存明文邀请码 |
| `created_by` | INT | 创建邀请码的管理员用户 ID |
| `used_by` | INT | 使用邀请码注册的用户 ID，未使用时为空 |
| `used_at` | TIMESTAMP | 使用时间，未使用时为空 |
| `is_active` | TINYINT | 是否有效，1 有效，0 失效 |
| `create_time` | TIMESTAMP | 创建时间 |
| `last_modified_time` | TIMESTAMP | 最后修改时间 |

## 权限规则

- 创建分组的用户自动成为 `ADMIN`。
- 使用邀请码加入分组的用户默认为 `USER`。
- 只有 `ADMIN` 可以进入人员管理、生成邀请码、调整成员权限。
- 一个分组至少保留一个 `ADMIN`。
- 管理员不能把当前分组最后一个 `ADMIN` 降级为 `USER`。
- 邀请码只能用于其所属分组。
- 邀请码默认一次性使用。

## 邀请码规则

- 邀请码由服务端生成随机字符串，不能由 `group_id` 直接编码得到。
- 邀请码展示给管理员用于复制或分享。
- 数据库只保存 `code_hash`，不保存明文邀请码。
- 用户提交邀请码时，服务端对输入值做相同哈希后查询。
- 邀请码使用成功后设置 `is_active = 0`，记录 `used_by` 和 `used_at`。
- 首版不要求多次使用、指定角色、过期时间；后续可扩展。

## 注册表单字段

### 创建分组

| 字段 | 必填 | 说明 |
|------|------|------|
| 分组名称 | 是 | 创建 `group.name` |
| 用户名/昵称 | 否 | 可复用微信信息或注册后在「我的」补充 |
| 账号密码 | 按登录方式决定 | 若保留账号密码登录，需要支持创建账号 |

### 加入分组

| 字段 | 必填 | 说明 |
|------|------|------|
| 邀请码 | 是 | 服务端校验后解析到对应分组 |
| 用户名/昵称 | 否 | 可复用微信信息或注册后在「我的」补充 |
| 账号密码 | 按登录方式决定 | 若保留账号密码登录，需要支持创建账号 |

## 成功标准

- 新用户可以不依赖手动写库完成分组创建和加入。
- 注册成功后用户拥有有效 token，并能正常进入首页。
- 创建分组用户的 `role` 为 `ADMIN`，加入分组用户的 `role` 为 `USER`。
- 邀请码无法通过猜测 `group_id` 加入分组。
- 已使用的邀请码不能再次使用。
- 管理员可以查看同组成员并调整权限。
- 服务端能阻止非管理员调用人员管理相关接口。
- 服务端能阻止一个分组失去最后一个管理员。
- 新增页面复用现有组件、管理页布局、loading/empty 状态和主题 token。

## 非目标

- 首版不支持跨分组切换。
- 首版不支持一个用户加入多个分组。
- 首版不支持邀请码多次使用。
- 首版不支持按邀请码指定加入者角色。
- 首版不做完整组织架构，仅管理当前 `group` 下的用户和角色。

## 待确认事项

- 注册页是否必须同时支持微信注册和账号密码注册，还是首版只接入微信注册。
- 注册成功后是否一定自动登录进入首页；当前推荐自动登录。
- 邀请码是否需要展示有效期；当前推荐首版不展示过期，只做一次性。
